# 帮助文档·深度设计规范

## 1. 核心理念
### 1.1 模块化思维
*   **概念**：文档本身也是由“块”组成的。
*   **应用**：一段文字、一张图、一个提示框，都是独立的拼图，可以随意组合但界限分明。
### 1.2 极简认知负荷
*   **原则**：不要让用户“阅读”，要让用户“扫描”。
*   **手段**：大留白、短句、动态图演示。

## 2. 页面结构与布局
*   **导语**：
    *   *规范*：**绝对不超过两行**。
    *   *内容*：一句话定义核心概念 + 一句话说明价值。
    *   *例*：“Block 是 Notion 的基本单位。它就像乐高积木，能让你构建出任何想要的内容。”

### 2.2 内容区
*   **分栏布局**：
    *   *左/中*：正文内容（占据 70%-80% 宽度）。
    *   *右*：悬浮目录，点击平滑滚动跳转。
*   **多平台切换**：
    *   *设计*：如果 Desktop/Mobile/Web 操作不同，使用 **Tab 选项卡** 切换内容，而不是写在一段话里混淆。

### 2.3 底部区
*   **反馈模块**：
    *   *文案*：Was this article helpful? ( 👍 / 👎 )
*   **相关推荐**：
    *   *逻辑*：基于当前阅读内容的“下一步”操作（如看了“什么是Block”，下一步推荐“如何移动Block”）。

## 3. 写作微观规范 
### 3.1 文本格式化
*   **UI 元素**：所有界面上的按钮、菜单名、选项，必须**加粗**。
    *   *例*：Click on **Settings & members**.
*   **快捷键**：使用`<kbd>`样式（类似键盘按键的视觉效果）。
    *   *例*：Press `cmd/ctrl` + `n`.
*   **链接**：
    *   *禁止*：不要用 "Click here" 或 "点击这里"。
    *   *规范*：将链接嵌入名词动词中。
    *   *例*：Read more about [database properties](link).

### 3.2 语调与句式 (Tone & Syntax)
*   **第二人称**：
    *   *规范*：全程使用“你（You）”，建立一对一对话感。
*   **祈使句**：
    *   *规范*：省略主语，直接动词开头。
    *   *对比*：
        *   错误：你应该点击右上角的按钮。
        *   正确：点击右上角的 **...** 按钮。
*   **禁止括号对照与语言原则**：
    *   *禁止括号对照*：严禁使用“标题 (Title)”或“设置 (Settings)”这种格式。
    *   *单一语言原则*：直接使用中文。
*   **比喻化**：
    *   *核心*：将抽象技术概念转化为物理世界概念。
    *   *Notion 案例*：Block = 乐高积木；Database = 文件柜。

### 3.3 特殊组件应用
*   **Callout**：
    *   🔵 **蓝色背景 + ℹ️ 图标**：补充说明/上下文。
    *   🟡 **黄色背景 + 💡 图标**：高阶技巧。
    *   🔴 **红色背景 + ⚠️ 图标**：警示/不可逆操作。
*   **Toggle**：
    *   *场景*：用于“如何恢复旧版本”这种不是所有人都需要的边缘操作。
    *   *目的*：保持页面主流程短小精悍。

## 4. 视觉与媒体规范
### 4.1 动态演示 (GIFs)
*   **尺寸**：宽度 100% 撑满内容区。
*   **时长**：3-8 秒。
*   **内容**：
    *   一次只演示**一个动作**。
    *   鼠标轨迹平滑，不要乱晃。
    *   **关键**：开始和结束画面要尽量一致，形成无缝循环 (Loop)。
*   **背景**：必须是干净的测试账号环境，不要出现杂乱的无关数据。

### 4.2 静态截图 (Screenshots)
*   **裁切**：
    *   *禁止*：截全屏（除非讲整体布局）。
    *   *规范*：只截取 UI 相关局部（Focus Area）。
*   **标注**：
    *   使用 Notion 品牌色的圆角矩形框选重点。
    *   极少使用文字标注在图片上（文字写在图片下方的 Caption 里，方便 SEO 和修改）。

### 4.3 图标 (Iconography)
*   **行内图标**：如果文案里提到了界面上的图标，文字后面要紧跟一个小图标。
    *   *例*：Click the `+` icon followed by the six dots `⋮⋮`.
